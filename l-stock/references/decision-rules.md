# Decision Rules

`l-stock` 的输出是交易决策支持，不是下单指令，也不保证结果。所有建议都必须保留条件、触发位和失效条件。

## 决策链

按这个顺序判断，不跳步：

1. 资金定方向：先看指数、ETF 规模变化、板块资金流向，判断市场是进攻、防守还是观望。
2. 情绪定阶段：再看涨停家数、炸板率、连板高度和龙头表现，判断情绪是启动、扩散、高潮、退潮还是修复。
3. 板块定主战场：只在资金和情绪共同支持的方向里选主线，避免孤立看个股。
4. 赔率定仓位：关注池个股必须同时有 `price`、`target`、`stop`。赔率低于 2:1 时禁止买入。
5. 技术定买卖点：买入必须等技术信号成立；持仓用 MA20/MA40 做纪律线。
6. 止损定生死：先处理失效，再讨论加仓或格局。

## 持仓规则

持仓动作由快照中的个股价格和均线触发：

- 收盘价跌破 MA40：`必须执行`，次日开盘一次性退出。
- 收盘价跌破 MA20 但未破 MA40：`条件执行`，先波段减仓；若 1-2 日不能快速收回 MA20，继续降风险。
- 未跌破 MA20/MA40：`观察等待`，继续持有并跟踪纪律线。

MA40 破位后不得摊低成本、不得补破位仓。重新纳入只能按关注池逻辑重建：重新给出 `price`、`target`、`stop`、技术信号和赔率。

## 关注池规则

关注池不是买入清单。只有满足以下条件，报告才允许给出条件买入：

- `price`、`target`、`stop` 可计算。
- `(target - price) / (price - stop) >= 2.0`。
- 技术信号成立，例如放量突破、回踩确认、重新站回关键均线。
- 市场方向、情绪阶段和板块主线不冲突。

赔率小于 2:1，或缺少目标价/止损价时，动作归入 `禁止动作` 或 `观察等待`。

## 仓位和现金

默认预备现金为 20%，来自 `state/preferences.yaml` 的：

```json
{
  "risk": {
    "reserve_cash_ratio": 0.2,
    "max_loss_per_trade_ratio": 0.02,
    "minimum_odds_ratio": 2.0
  }
}
```

没有明确信号时，不动用预备现金。条件买入默认只能用试探仓，当前报告渲染为 `1/4 到 1/3`；若市场或情绪不支持，宁可不交易。

## 报告表达要求

每个建议必须回答：

- 动作：退出、减仓、持有、条件买入、禁止买入。
- 触发位：MA20、MA40、目标价、止损价或具体技术信号。
- 仓位：退出、减仓、试探仓、保留现金。
- 失效条件：跌破、不能收回、赔率不足或数据不足。

不要写“看好”“有机会”“可关注”这类没有触发条件的空话。
